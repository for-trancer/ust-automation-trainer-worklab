# MANUAL TESTING (Predefined Test Cases)

- Types (Functional and Non Functional)

* Functional Testing

1.Acceptance Testing : Confirms the product meet buiseness requirements (performed by end users / clients)
2.Integration Testing : Ensure multiple module works together
3.Unit Testing : Individual Components
4.Functional Testing : Tests the entire system
5.Retesting : 
6.Regression Testing : 
7.Smoke Testing : Running tests on most critical functions(eg.after the bug fix/built release)
8.Sanity Testing : Tested on stable builds and ensures that the built is working

* Non Functional Testing

1.Performance Testing : Checks speed,responsiveness and stability
2.Stress Testing : Tests how the system behaves under high load
3.Usability Testing : Ensures the app is easy to use and user-friendly

- Testing Approaches

* Black Box : No Source Code Visible 

	a.Equiliance Partition : Dividing input data into valid and invalid partitions (password length)
	(0-7 : invalid , 8-12 : valid , 13+ : invalid)

	b.Boundary Value Analysis : In scope and out of scope

	c.State Transition Testing : Checking how the system changes from one state to another(ATM: Enter pin to account menu or access denied after 3 failed attempts)

	d.Decision Table Technique : Visual representation of specific action

* White Box : Visible Source Code

	a.Statement Coverage : Every executable line of code is atleast covered once in testing (if else)

	b.Branch Coverage : Every possible outcome is from each decision point is executed

	c.Path Coverage : Ensures every execution path is tested
	 

* Gray Box : Combination of Black And White
